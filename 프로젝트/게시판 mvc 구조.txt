[각 계층별 값 넘기고 받는 구조]

뷰<->컨트롤러<-> 서비스<-> 매퍼


*매개변수: 호출한 클래스에서 보내는(input) 값
*리턴값: 호출한 클래스에서 받는(output) 값
*메소드 내에서 넘겨야할 Dto가 있으면 각 메소드 내부에서 생성한다.
ex) 컨트롤러에서 서비스에게 매개변수로 CUDInfo와 BoardDto를 넘겨야한다.
--> 컨트롤러 메소드 내부에서 CUDInfo와 BoardDto 생성



뷰 : POST/GET 방식으로 URL 지정하여 컨트롤러 메소드 호출
컨트롤러 : 서비스 메소드 호출
서비스 : 매퍼 메소드 호출

컨트롤러 메소드의 매개변수 : 뷰에서 넘긴 값
컨트롤러 메소드의 리턴 값 :  뷰에서 받을 값

서비스 메소드의 매개변수 : 컨트롤러에서 넘긴 값
서비스 메소드의 리턴 값 : 컨트롤러에서 받을 값

매퍼의 매개변수 : 서비스에서 넘긴 값
매퍼의 리턴 값 : 서비스에서 받을 값



[게시판 구조 정리]

1.뷰는 클라이언트에서 입력한객체(BoardDto)를 컨트롤러에 넘김
2.컨트롤러는 (뷰가 넘긴)BoardDto를 받아와서 CUDInfoDto를 생성하고
 서비스의 메소드 호출(BoardDto,CUDInfoDto를 넘기고 결과값인 BoardDto를 받음)
3.서비스는 (컨트롤러가 넘긴) CUDInfoDto와 BoardDto를 받아와서 
 CUDInfoDto의 메소드호출(BoardDto 넘겨서 BoardDto의 필드값셋팅),
 매퍼의 메소드 호출(BoardDto 넘겨서 BoardDto를 DB에 저장)
4. 매퍼는 받아온 BoardDto를 가지고 쿼리 실행
